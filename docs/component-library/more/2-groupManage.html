团队管理

团队管理

创建团队

登录平台后，你可以点击左侧导航「 + 」按钮，选择「 团队 」，输入你的团队名称、团队标识、头像、PI姓名等信息，邀请成员加入团队，点击创建即可。

TIP

  • 同一个账号可以创建多个团队，并且加入多个团队。
  • 创建团队的账号，默认为该团队的所属者，拥有团队管理最高权限。

切换团队

当你参与到多个团队时，可以通过「 切换团队 」功能快速切入目标团队。

登录平台后，点击右上角的个人头像，点击团队管理，展开你所在的所有管理团队，选择需要切换的团队名称即可进入管理。

退出团队

如果你后续不再参与该团队协作，你可以选择退出团队。

点击右上角个人头像，点击个人主页，进入「 已加入团队列表 」，选择需要退出的团队，点击「 退出 」按钮即可。

TIP

  • 团队所属者不能直接退出团队，需先移交团队，才能退出团队。

解散团队

如果你是团队所属者，想要解散团队，进入到团队管理，跳转至「 团队设置 」，点击解散团队。

TIP

  • 在解散团队之前，需先给所有团队项目选择新的所属者，才能解散团队。
  • 解散团队后，该团队的所有内容将被删除，且不可恢复，请慎重操作。

成员管理

作为团队管理角色的你，可以在这里查看所有成员信息，并进行成员管理相关操作，让团队协作井井有条～

  • 邀请新成员加入团队
  • 为团队成员进行角色分配、项目权限的设置
  • 移出团队
  • 查看、管理成员加入审批

邀请成员加入团队

团队内所有成员都可以邀请新成员加入团队，管理者可在管理界面邀请新成员加入。支持通过邮箱邀请平台未注册用户加入。邀请时可以同时选择该成员的团队角色，也可以邀请加入团队后调整，支持单个新成员邀请和一次性批量邀请。

功能入口：

  1. 点击右上角个人头像，点击「 邀请成员 」。
  2. 进入团队管理--成员管理页，单击页面右上角「 邀请成员 」。

操作流程：

  1. 在「 邀请成员 」弹窗内，输入用户账号信息（邮箱地址、用户名、姓名）搜索加入，并为成员设置角色。
  2. 点击 「 邀请 」完成。

设置成员角色权限

你可以通过搜索找到对应成员进行单个成员的角色分配与管理。

操作流程：

  1. 进入「 成员管理 」页，找到需要分配、修改角色的成员。
  2. 点击「 修改角色 」，下拉选择角色名称，点击确认完成。

移出成员

团队所有者、团队管理员可以将成员移出团队。进入成员管理页，找到需要移出的成员，点击「 移出团队 」。

等待管理员审批

系统用户搜索团队申请加入后，需要经过管理员审批。管理员可以在「 等待管理员审批 」页查看所有申请信息。

在「 等待管理员审批 」中找到需要审批的新成员，点击「 同意 」或「 拒绝 」用户加入团队。

TIP

  • 如果需要批量完成审批，你可以勾选“姓名”左侧的方框，单击页面右上角批量「 拒绝 」或「 同意 」来完成快速审批。
  • 当申请被拒绝时，系统将会通知提出加入申请的成员。

等待用户同意

在向新成员发出邀请后，管理员可以在「 等待用户同意 」页查看所有邀请信息。

对于受邀成员未处理的邀请，可以在这里进行重新邀请或撤销邀请的操作。

TIP

  1. 列表按照新成员发起邀请的时间排序展示，最近发起的邀请用户最先展示。
  2. 如果多次邀请同一个用户，只显示最近一次的邀请时间。
  3. 支持输入该用户名、姓名或邮箱信息进行查询。
  4. 如果需要批量完成审批，你可以勾选“姓名”左侧的方框，单击页面右上角批量「 重发邀请 」或「 撤销邀请 」来完成快速审批。

仓库管理

在这里，您可以构建团队仓库的整体位置层级架构。团队仓库管理员负责统一构建和分配仓库空间，而项目组仓库负责人可以在分配的范围内独立管理项目组的元件样品，实现团队与项目组分层管理的高效协作。 建议按照自上而下的顺序创建位置层级（例如：冰箱 → 第一层 → 第二层 → … → 盒子 → 容器/板）。建立位置层次结构后，你可以进一步创建和配置盒子、板子和容器。

创建位置

方式一：单个位置创建

在仓库管理页面，点击「 + 」按钮，选择位置，点击「 创建位置 」选项。

位置类型：给当前创建的位置选择位置类型（如房间、冰箱、层）。 位置编码：可以使用位置类型定义的前缀和顺序编号（如 R001、R002）自动生成条形码，也可以手动输入唯一编码。 位置名称：输入位置名称。 上层位置：指定当前创建位置的父级（例如，将冰箱添加到房间中）。如果创建的是顶级位置（如房间或实验室），则直接选择“顶层”。 位置状态：设置是否启用该位置。 描述：输入对该位置的一些描述信息。

方式二：批量导入创建位置

在仓库管理页面，点击「 + 」按钮，选择位置，点击「 批量导入位置 」选项。

在批量导入页面，通过选择要批量创建的位置类型，再点击下载模板文件，按照模板填写要求完成录入后，再将填写好的模板上传系统批量创建。

TIP

  • 创建位置前，你必须确保有可使用的位置类型，可先在数据模板配置。
  • 若选择的位置类型有配置自定义字段，则根据设置进行输入即可。
  • 若你只是项目仓库管理员则无位置操作权限。

创建盒子

方式一：单个盒子创建

在仓库管理页面，点击「 + 」按钮，选择盒子，点击「 创建盒子 」选项。

盒子类型：给当前创建的盒子选择盒子类型（如9x9 盒、10x10 盒）。 盒子编码：可以使用盒子类型定义的前缀和顺序编号（如 box001、box002）自动生成条形码，也可以手动输入唯一编码。 盒子名称：输入盒子名称。 容器类型：给当前创建的盒子选择使用的容器类型。 存储位置：给当前创建的盒子选择存储的位置。 元件类型：给当前创建的盒子选择储藏样品的元件类型。 样品类型：设置盒子用于储藏日常样品还是备份样品。 盒子状态：设置是否启用该盒子。 描述：输入对该盒子的一些描述信息。

方式二：批量盒子创建

在仓库管理页面，点击「 + 」按钮，选择盒子，点击「 批量导入盒子 」选项。

在批量导入页面，通过选择要批量创建的盒子类型，再点击下载模板文件，按照模板填写要求完成录入后，再将填写好的模板上传系统批量创建。

TIP

  • 创建盒子前，你必须确保有可使用的盒子类型和容器类型，可先在数据模板配置。
  • 若选择的盒子类型有配置自定义字段，则根据设置进行输入即可。

创建板子

方式一：单个板子创建

在仓库管理页面，点击「 + 」按钮，选择板子，点击「 创建板子 」选项。

板子类型：给当前创建的板子选择盒子类型（如9x9 盒、10x10 盒）。 板子编码：可以使用板子类型定义的前缀和顺序编号（如 plate001、plate002）自动生成条形码，也可以手动输入唯一编码。 板子名称：输入板子名称。 板子类型：给当前创建的板子选择使用的容器类型。 存储位置：给当前创建的板子选择存储的位置。 元件类型：给当前创建的板子选择储藏样品的元件类型。 样品类型：设置板子用于储藏日常样品还是备份样品。 板子状态：设置是否启用该板子。 描述：输入对该板子的一些描述信息。

方式二：批量板子创建

在仓库管理页面，点击「 + 」按钮，选择板子，点击「 批量导入板子 」选项。

在批量导入页面，通过选择要批量创建的板子类型，再点击下载模板文件，按照模板填写要求完成录入后，再将填写好的模板上传系统批量创建。

TIP

  • 创建板子前，你必须确保有可使用的板子类型和容器类型，可先在数据模板配置。
  • 若选择的板子类型有配置自定义字段，则根据设置进行输入即可。

创建容器

功能入口：

  1. 在仓库管理页面，点击「 + 」按钮，选择容器，点击「 创建容器 」选项。
  2. 在仓库管理页面，点击「 + 」按钮，选择容器，点击「 批量创建容器 」选项。
  3. 找到创建容器的盒子，进入到盒子详情页面，选择需要创建容器的孔位，点击「 创建容器 」按钮。

操作流程：

  1. 选择需要创建容器的盒子。
  2. 选择空的孔位。
  3. 点击「 下一步 」填写容器名称和容器编码。若使用容器类型定义的前缀和顺序编号（如 tube001、tube002）自动生成条形码,则将该字段留空。反之，若使用自定义编码，则输入唯一编码。

TIP

  • 创建板子时，系统会根据选择的容器类型自动生成容器。你可以自定义容器的名称和编码。

移动到

你可以将位置、盒子、板子、容器从当前位置移动到另一个位置。找到需要移动的位置、盒子、板子、容器，点击「 移动到 」按钮，选择新的存储路径。

填充样品

  1. 进入板盒页面，通过与左侧的网格视图或右侧的容器列表进行交互来选择板盒中的容器，然后选择「 填充容器 」。你可以通过此操作选择一个或多个容器进行填充。
  2. 指定是从已有样品中选择进行制备填充，还是从元件中选择制备填充。
  3. 填写准备制备样品的浓度、体积。若是从已有样品中选择制备，请填写取样样品剩余体积以及是否进行了冻融。
  4. 可指定制备的样品是否用于某个样品申请单。若指定给样品申请单，则该样品不能进行操作。用户申请完成后，系统自动将样品发送到用户个人仓库中。

样品出入库

仓库管理员可以将样品从指定位置出库，使用完成后再入库放回原位置。

功能入口：

  1. 在仓库列表，切换到“容器视图”，选择需要出入库的样品。
  2. 进入板盒页面，从容器列表选择样品出入库。

备份启用

当日常样品不够用时，你可以选择备份样品申请转为日常样品使用或制备新的样品。

功能入口：

  1. 在仓库列表，切换到“容器视图”，选择需要启用的备份样品。
  2. 进入板盒页面，从容器列表选择备份样品发起申请。

操作流程：

  1. 进入板盒页面，通过与左侧的网格视图或右侧的容器列表进行交互来选择板盒中的样品，然后选择「 备份启用 」。你可以通过此操作选择一个或多个备份样品进行启用。
  2. 指定是用备份样品制备日常样品，还是直接将备份样品转为日常样品。
  3. 填写准备制备样品的浓度、体积。
  4. 填写备注信息。

样品销毁

当样品无法使用或已在实体仓库中清理时，你可以选择申请将样品销毁。

功能入口：

  1. 在仓库列表，切换到“容器视图”，选择需要销毁的样品。
  2. 进入板盒页面，从容器列表选择样品进行销毁。

导出样品

方式一：选择位置层级导出

在仓库列表，切换到“位置视图”，你可以选择某一层的位置（如仓库、房间、冰箱等），系统将自动导出该层位置下包含的所有样品信息。

方式二：选择板盒导出

在仓库列表，选择盒子、板子，系统将导出所选盒子、板子内的所有样品信息。

方式三：选择容器列表导出

在仓库列表，切换到“容器视图”，选择你到要导出的样品信息。

仓库视图切换/筛选/排序

视图切换

在仓库管理界面，你可以通过下拉菜单在“团队/项目组视图”、“位置视图”、“板盒视图”和“容器视图”之间切换。

  • 团队/项目组视图：支持在团队整体仓库位置层级视图和各项目组分配的仓库视图之间切换。团队视图展示整个仓库的结构和状态，项目组视图仅显示分配给项目组的空间及其数据，便于分层管理。
  • 位置视图：显示位置及其包含的子位置和板盒数据，适合层级结构查看和管理仓库。
  • 板盒视图：显示所有的板子、盒子数据，适合快速浏览和操作具体盒子和板子。
  • 容器视图：显示所有的容器数据，包括已填充样品和未填充的样品，适合快速浏览和操作具体容器。

筛选

在仓库管理界面，你可以通过筛选功能查看指定的数据类型，快速浏览各类数据。

  • 类型筛选：在位置路径下，可以按位置、板盒、容器等分类筛选数据，方便用户根据特定分类快速查找。
  • 表头筛选：在仓库列表的表头，点击各列的筛选图标，选择筛选条件，快速筛选出符合条件的数据。
  • 搜索框筛选：在仓库管理顶部的搜索框，输入关键词进行搜索，快速找到包含该关键词的位置、板盒、容器。

TIP

  1. 三种搜索方式之间是and关系。即多种方式检索条件存在时，页面显示条件都符合的数据。
  2. 点击【重置】按钮，清空页面检索条件，刷新页面。

排序

在仓库列表，你可以根据个人需求，选择按照顺序对仓库列表进行排序以及对仓库列表的各列进行排序和显示控制。

  • 排序数据：根据个人需求，选择按照名称、创建时间、修改时间等排序方式对数据进行排序，方便整理和查看。
  • 列排序与显示：根据个人需要，你可以对仓库列表的各列进行排序，还可以自定义显示字段设置，选择哪些列需要显示或隐藏，以便更清晰地查看和管理仓库数据。

仓库分配

为团队项目分配仓库空间。各项目组仓库负责人可以在分配的空间里管理项目组的实体元件样品。

  1. 进入“团队管理”-“项目管理”页面。在项目列表中，找到需要分配仓库的项目，点击对应的“分配仓库”按钮，弹出分配的窗口。
  2. 在弹窗的左侧，显示当前可分配的仓库位置空间列表。通过勾选框选择需要分配的位置层级。
  3. 勾选需要分配的位置空间后，点击中间的【右箭头】按钮（“>”），将位置移动到右侧的“已分配”区域。如果需要取消分配，勾选右侧的位置后，点击左箭头按钮（“<”），将位置移回左侧的可分配列表。
  4. 确认分配完成后，点击弹窗右下角的【确认】按钮，保存分配结果。

TIP

  • 项目组在分配的空间内自行构建的位置层级（如新增子位置、板子、盒子等），这些内容不会显示在“分配仓库”界面，操作者仅能看到原始分配的空间信息。
  • 当某个项目被删除时，该项目仓库内的数据（如位置、盒子、板子等）将被清空。被清空的位置空间将自动释放，允许重新分配给其他项目组。
  • 如果某个位置空间已分配给项目组，并且在该空间内已有数据（如位置、盒子、板子等），在重新分配之前，必须先清空（即删除）相关数据。

审批流程配置

用于为元件各种申请类型设置审批流程的模块。通过该模块，管理员可以根据不同的业务需求，灵活定义和管理申请的审批流程，确保申请按照预定的步骤得到有效处理。

  1. 进入到“审批流程配置”页面，选择需要配置审批流程的申请类型，点击【流程设计】按钮进入配置。包括样品入库、样品申请、样品销毁、元件发布、备份启用、样品寄存、清空寄存。

  2. 添加审批节点：点击【添加节点】按钮，为流程添加审批节点，定义每个节点的审批节点名称、审批人和审批动作。

  • 设置抄送人：管理员点击任一审批节点，在右侧抄送人一栏中点击【添加抄送人】按钮，选择抄送人。抄送人可以在以下三个环节接收通知。

  • 流程发起时：在该环节设置抄送，即当有用户发起此审批时，相应的抄送人就会收到通知。

  • 流程中间节点：在该环节设置抄送，当节点审批人审批后，抄送人会收到通知。

  • 流程结束时：在该环节设置抄送，即当此审批流程结束时，相应的抄送人就会收到通知。

  • 关联审批表单：设置审批节点时，可以选择关联特定的审批表单。当审批人处理该节点的审批请求时，需填写关联表单中的信息。

  • 自动分配库位及确认：在样品入库和备份启用流程中，必须有一个审批节点勾选“是否入库确认并分配库位”和“是否备份启用确认并分配库位”。用于样品入库和备份启用操作的确认和库位分配。审批通过后，系统将自动完成样品的入库，并为其分配合适的库位。

  1. 完成所有设置后，点击右上角【保存】。配置完成的审批流程将在相应的申请类型中自动生效。

TIP

  • 多人审批时系统默认或签，审批人为空时自动通过。

团队项目角色权限配置

用于管理团队项目中的角色及其对应的权限，确保不同角色在项目、文件夹、元件等层级上拥有合适的访问和操作权限。

平台默认提供了“项目管理员”和“项目成员”两个角色。如果这些角色无法满足你的需求，你还可以自定义添加新的角色，并配置其权限，以更好地适应项目的管理要求。

团队设置

在「团队设置」页，可以进行团队相关的基础设置，包括团队信息、解散团队的操作。

TIP

  • 团队设置页属于仅所属者有操作的权限。
  • 解散团队时，需为每个团队项目选择新的所属者。

团队权限说明

团队权限包含 4 种角色：所属者、团队管理员、仓库管理员、团队成员，权限操作遵循“所属者 > 团队管里员 > 仓库管理员 >团队成员”的原则；

不同的团队角色，只会影响团队项目管理、成员管理、仓库管理、数据模板配置等功能的操作权限，但不影响平台基础功能的使用，比如元件管理、个人主页配置等。
